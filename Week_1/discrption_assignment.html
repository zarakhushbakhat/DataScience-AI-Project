Titanic Project — Setup & Upload Steps

This page lists the complete step-by-step workflow you requested: install Python, install Jupyter, install libraries, prepare the Titanic dataset, run the notebook, and upload the project to GitHub. Follow each step in order.

1. Install Python

  1. Open python.org/downloads and download the latest stable Python 3.x installer for your OS.
  2. Run the installer and be sure to check “Add Python to PATH” during installation.
  3. Verify installation in a terminal / command prompt: 
    python --version
    or on Windows you can try: 
    py --version

2. Install Jupyter Notebook

  1. Open your terminal or command prompt.
  2. Install Jupyter with pip: 
    pip install notebook
  3. Launch Jupyter in the folder where you will work: 
    jupyter notebook
    This opens a browser where you can create and run notebooks (`.ipynb`).

3. (Optional) Install VS Code and Extensions

  1. Download VS Code from code.visualstudio.com.
  2. Install these extensions from the Extensions panel in VS Code:
    • Python (by Microsoft)
    • Jupyter (by Microsoft)
    • Pylance (optional — improved IntelliSense)
  3. You can open and run `.ipynb` notebooks directly inside VS Code after selecting the proper Python kernel (top-right of the notebook view).

4. Install Required Python Libraries

Install the libraries used in the Titanic example and general data work:

pip install pandas numpy matplotlib seaborn tabulate

If you see a ModuleNotFoundError when running code, install the missing library with pip install <library-name>.

5. Get the Titanic Dataset

  1. You can use the built-in Titanic dataset from seaborn or download a CSV from Kaggle:
    • Kaggle Titanic: https://www.kaggle.com/c/titanic/data
  2. If you download titanic.csv, save it to your project folder (e.g. F:\semester 7\Titanic-Project\titanic.csv).

6. Create Your Project Files

  1. Open Jupyter or VS Code and create a new notebook named titanic_analysis.ipynb.
  2. Add code cells to load the data, show column descriptions, preview the table, and add basic plots — run cells to verify outputs.
  3. Save your notebook in the project folder.

7. Run & Test

  1. Start Jupyter in the project folder: 
    jupyter notebook
  2. Open titanic_analysis.ipynb and run all cells. Confirm outputs (data preview, plots) show correctly.

8. Prepare to Upload to GitHub

  1. If you don't have a GitHub account, create one at github.com.
  2. Create a new repository on GitHub (example name: Titanic-Data-Analysis).
  3. Do NOT initialize with README on GitHub if you already have files locally (optional).

9. Upload Local Project to GitHub (Command Line)

Open your terminal in your project folder and run:

git init
git branch -M main
git add .
git commit -m "Initial commit: Titanic analysis"
git remote add origin https://github.com/yourusername/Titanic-Data-Analysis.git
git push -u origin main

If Git asks for credentials when pushing, enter your GitHub username and a Personal Access Token (PAT) as the password (GitHub no longer accepts account passwords for command-line push).

10. Alternative: Upload Manually via GitHub Website

  1. Go to your repository page → click Add file → Upload files.
  2. Drag & drop your notebook and small files. Commit changes.
  3. Note: GitHub web UI limits single file uploads (about 25 MB per file). For large files use Git LFS or cloud storage.

11. If You Have Large Files

  1. For files over 100 MB, do not push to GitHub. Instead upload large datasets to Google Drive / Dropbox / OneDrive and include download links in your README.
  2. For files under 100 MB but large, use Git LFS: 
    git lfs install
git lfs track "*.csv"
git add .gitattributes
git add <large files>
git commit -m "Track large files with LFS"
git push -u origin main

12. Create a README

  1. Create a file named README.md with a short project description, dataset links (if hosted externally), and how to run the notebook.
  2. Add, commit, and push the README to GitHub: 
    git add README.md
git commit -m "Add README"
git push
